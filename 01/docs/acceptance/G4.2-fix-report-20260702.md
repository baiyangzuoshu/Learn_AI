## 4. 关键代码说明

### connect() 如何被接入调用链

```typescript
// cdp-manager.ts 新增方法
async ensureConnected(timeoutMs?: number): Promise<RuntimeStatus> {
  if (this.isConnected()) {
    // 已连接 → ping 验证存活
    try { await this.ping(); return this.getStatus(); }
    catch { await this.disconnect(); } // 断线 → 重连
  }
  // 未连接 → 尝试 connect()（可指定超时）
  const orig = this.config.timeoutMs;
  if (timeoutMs) this.config.timeoutMs = timeoutMs;
  try { return await this.connect(); }
  finally { this.config.timeoutMs = orig; }
}
```

### runtime_status 的调用链

```
handleRuntimeStatus()
  → cdpManager.ensureConnected(1500)  // 1.5s 快速超时
    → connect()                        // 真实 CDP 连接尝试
      → CDP.List() → 选目标 → CDP() → Runtime.enable() → 验证 cc.director
  → if connected: evaluateSafe(buildStatusExpression()) → 返回场景/分辨率
  → if !connected: 返回 { status: "disconnected", error: "..." }
```

### 其他 4 个 handler 的调用链

```
handleRuntimeXxx()
  → cdpManager.ensureConnected()  // 默认 5s 超时
  → if !connected: 返回 RUNTIME_UNAVAILABLE
  → if connected: evaluateSafe(expression) → 返回真实数据
```

### 未连接 / 连接成功 / 连接失败 / 断线后的行为

| 状态 | runtime_status | 其他 4 个 tools |
|---|---|---|
| 从未连接 | ensureConnected → connect() 失败 → `disconnected` | ensureConnected → connect() 失败 → `RUNTIME_UNAVAILABLE` |
| 连接成功 | ensureConnected → connected → 读取场景信息 | ensureConnected → connected → 执行白名单脚本 |
| 连接失败 | 错误消息写入 status.error，不崩溃 | 错误消息写入 RUNTIME_UNAVAILABLE，不崩溃 |
| 断线后 | ensureConnected → ping 失败 → disconnect → reconnect 尝试 | 同左 |

### 为什么这样设计

1. **懒连接而非启动时连接**：MCP Server 启动时 Cocos 预览可能还没启动。懒连接避免启动失败。
2. **ensureConnected 统一入口**：避免 5 个 handler 各自实现不一致的连接逻辑。
3. **runtime_status 用 1.5s 快速超时**：保证 G4-01 的 2 秒要求（1.5s 连接 + 0.5s 余量）。
4. **其他 handler 用 5s 超时**：允许更长的连接时间，因为用户期望这些操作需要连接。
